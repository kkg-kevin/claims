Architecture & conventions
- **Single-page UI**: `App.tsx` is the primary UI module — it contains types, mock data and the majority of views. When integrating, consider splitting large views into feature folders under `src/app/`.
- **Types & mock data**: Core domain types (`Course`, `Student`, `Assignment`, etc.) are defined at the top of `App.tsx`. For larger work, move them to `src/lib/types.ts` and import.
- **Components**: Re-usable UI elements live under `src/app/components/ui`. Follow existing naming/props patterns and keep components small and well-typed.
- **Styling**: Tailwind + a local `theme.css` are used. Edit colors and tokens in `src/styles/theme.css` (and `default_shadcn_theme.css` if present).
- **Assets**: The Vite plugin resolves `figma:asset/<file>` to `src/assets/<file>` (see [vite.config.ts](vite.config.ts#L1-L80)). Put static images in `src/assets`.

Development notes
- React is listed as a peer dependency in `package.json`. If the app errors complaining about missing `react`/`react-dom`, install them locally:

```bash
pnpm add react react-dom
```

- Add dependencies with `pnpm add <pkg>` and dev-only libs with `pnpm add -D <pkg>`.
- To preview a production build locally: `pnpm exec vite preview` after `pnpm build`.
